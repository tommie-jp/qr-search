# お絵かきレイヤ計画 — セッション内レイヤ (Ver2.x)

お絵かき (docs/34, 36) にレイヤを足す。**方式 (a): セッション内限定**。
描いている間だけレイヤがあり、保存 (書き出し) は従来どおり 1 枚の
WebP/PNG に平坦化する。fabric の JSON は保存しない — 再編集で
レイヤが復元される方式 (サイドカー保存) は本計画の対象外で、
必要になったら別計画に切る。

## 1. なぜ・何ができるか

- **下描き → 清書**: レイヤ 1 に下描き、レイヤ 2 に清書。書き出す前に
  下描きレイヤを非表示にすれば、清書だけの画像になる
- **写真注釈の仕分け**: 寸法の注釈と目印の注釈を別レイヤに分け、
  片方だけ消したり非表示にしたりできる
- **誤消去の防止**: 消しゴム・選択は**アクティブレイヤにしか効かない**ので、
  描き込んだ注釈を守ったまま別レイヤを直せる

できないこと (仕様として割り切る):

- 保存後の再編集でレイヤは戻らない (背景画像として開く従来どおりの挙動)
- レイヤの並べ替え・追加・削除はしない (固定 3 枚)
- レイヤ間のオブジェクト移動はしない

## 2. 仕様

- レイヤは**固定 3 枚** (レイヤ 1 が最背面、レイヤ 3 が最前面)。
  背景画像 (`backgroundImage`) はさらにその下で、従来どおりレイヤ外
- 開いた直後のアクティブは**レイヤ 1**。既存の使い方 (レイヤを意識しない)
  なら、全部がレイヤ 1 に載って従来と同じに振る舞う
- 各レイヤは**表示/非表示**を切り替えられる。ただし**アクティブレイヤは
  非表示にできない** (見えない場所に描く事故を作らない)
- 新しく描いたもの (線・図形・文字・塗り・モザイク) は
  すべてアクティブレイヤに載る
- **消しゴム・選択はアクティブレイヤ限定**。非表示レイヤには何も効かない
- 書き出しは**見たまま (WYSIWYG)**: 非表示レイヤは画像に入らない。
  これが「下描きを外して清書だけ出す」の実現手段そのもの
- 「全消し」は従来どおり全レイヤを消す (1 手として戻せる)。
  レイヤ別の全消しは要望が出てから
- 取り消し履歴は従来どおり描画内容だけを対象にする。レイヤの切り替え・
  表示/非表示は道具や色の変更と同じ「見る側の状態」で、履歴に積まない

## 3. 実装の形

### 3-1. データモデル — Group は使わない

fabric はもともと全描画物を個別オブジェクトの z 順で持っている
(`preserveObjectStacking: true` 設定済み)。レイヤは
**各オブジェクトの拡張プロパティ `layer` (1〜3)** で表し、
canvas のオブジェクト列を「レイヤ 1 の帯 → 2 の帯 → 3 の帯」に保つ。

- Group 化はしない。自由描画のパスは canvas 直下に足されるので Group への
  移し替えが要るし、@erase2d の ClippingGroup とも重なって扱いが濁る。
  `erasable` と同じ「拡張プロパティ + 走査」の流儀が既存コードと揃う
- スナップショットは `fc.toObject(["erasable", "layer"])` に広げるだけで、
  undo/redo は既存機構 (docs/34 §5) にそのまま乗る
- `object:added` で (履歴の erasable 既定付与と同じ場所で) `layer` に
  アクティブレイヤを刻み、**自分の帯の末尾へ挿し込む** (`moveObjectTo`)。
  挿し込み位置の計算は純関数に切る (§5)。図形ドラッグ中の仮オブジェクトも
  同じ経路を通るが、確定時に 1 手として積む既存の仕組みは変わらない

### 3-2. レイヤ状態の当て直し — 1 つの純関数に集約

オブジェクトごとの働きは既に 3 つのフラグで制御している:

- `visible` — 描く/書き出すか
- `erasable` — 消しゴムが消せるか (@erase2d が見る)
- `selectable` / `evented` — 選択道具が掴めるか

これらを **(そのオブジェクトの layer, アクティブレイヤ, 非表示集合, 道具) →
フラグ一式** の純関数 `layerFlags()` で導出し、当て直しの口を 1 つにする:

```text
visible    = layer が非表示集合に無い
erasable   = layer === アクティブ && visible
selectable = 道具が select && layer === アクティブ && visible
```

当て直すタイミングは 3 つ — レイヤ状態の変更時・道具の切り替え時
(既存の `forEachObject` ループを置き換え)・履歴からの復元後
(`applyEntry` 内。スナップショットに古い visible/erasable が焼き込まれて
いても、ここで現在のレイヤ状態に揃うので食い違いが残らない)。

背景画像は従来どおり `erasable: false` / `evented: false` のままレイヤ外。

### 3-3. 既存道具との相互作用

- **塗りつぶし・モザイク** (ラスタ道具) は `lowerCanvasEl` の画素 = 表示中の
  合成結果を読む。つまり「見えているものに対して効き、結果はアクティブ
  レイヤに載る」。非表示レイヤの下の色は拾わない — WYSIWYG と同じ意味論
  なので仕様とする
- **消しゴム**は `erasable` の導出 (§3-2) だけでレイヤ限定になる。
  @erase2d は erasable が真のものしか消さない (docs/34) ので追加実装なし
- **選択**はアクティブレイヤ限定なので、レイヤをまたぐ複数選択
  (ActiveSelection) は発生しない。帯の z 順を壊す経路を作らない
- **文字・図形・ペン**は「足したものにアクティブレイヤを刻む」以外は不変

## 4. UI — レイヤボタン + 小パネル

ツールバー (横スクロール、道具 11 個 + 色・太さ・履歴・ズーム) には
これ以上並べられないので、**「レイヤ 1/2/3」表示のボタンを 1 つ**足し、
押すと小さなパネル (ポップオーバー) を開く。

- パネルは 3 行。各行 = タップでアクティブに / 右端に表示トグル (目のアイコン)
- アクティブ行の目は無効化 (§2 のルール)。行にはそのレイヤの
  オブジェクト数を小さく添える (どこに何があるか迷子にならないため)
- 描画・道具選択でパネルは閉じる。暗い覆いの上なのでツールバーと同じ
  自前配色 (DrawToolbar の BUTTON_* と揃える)
- 44px タップ領域・アイコンは線画 + currentColor の既存作法 (docs/31) に従う

## 5. 純粋ロジック (TDD 対象) — `src/lib/draw/layers.ts`

- `LAYER_COUNT = 3` と型 (`LayerId`, `LayerState = { active, hidden }`)
- `layerFlags(layer, state, tool)` → `{ visible, erasable, selectable }` (§3-2)
- `insertionIndex(layers, target)` — オブジェクト列の layer 並びから、
  target レイヤの帯の末尾位置を返す (§3-1 の挿し込み)
- `toggleHidden(state, layer)` — アクティブは非表示にできない規則ごと実装
- `setActive(state, layer)` — アクティブ化したレイヤを非表示集合から外す

いずれも fabric を触らない列・集合操作なので、既存の `history.ts` /
`shapes.ts` と同じくユニットテストを先に書く (AAA)。

### ブラウザ検証の記録 (Phase 4 — ローカル一巡・済)

Playwright + テスト認証の dev サーバ (:3001) で、下 canvas の画素を
帯ごとに実測して確認した (各レイヤの線を y=0.3/0.5/0.7 に引き分けた)。

- **レイヤ刻印**: アクティブを 1→2→3 と切り替えて線を引くと、パネルの
  オブジェクト数が `{1:1,2:0,0}`→`{1,1,0}`→`{1,1,1}` と、引いた先の
  レイヤだけ増える。`object:added` の刻印と帯挿入が効いている
- **非表示 = 書き出し対象外 (WYSIWYG)**: レイヤ 1 を隠すとその帯の
  非白画素が 2053→0、他帯は不変。1+2 を隠すとアクティブの 3 だけ残り、
  戻すと完全に復元。`exportImage` は同じ可視オブジェクトを描くので、
  隠したレイヤは画像に入らない
- **消しゴムはアクティブレイヤ限定**: アクティブ 2 で消しゴムをレイヤ 1 の
  線の上をなぞっても、その帯は 2053 のまま無傷。アクティブ (2・3) の帯は
  0 に消える。`erasable` フラグのレイヤ限定が効いている
- **アクティブは隠せない**: パネルのアクティブ行の目トグルは `disabled`
- **undo/redo**: 消した線が undo で戻り (帯 0→2150)、redo で再び消える。
  復元後に applyLayerState が走り、戻した線も正しく表示される
- **外側クリックで閉じる**: パネルは背景の覆いを押すと閉じる
- ランタイムエラー 0
- **実機 iPhone**: パネルのタップ・2 本指ジェスチャとの共存を確認済み

## 7. 難所・注意

- **スナップショット互換**: `layer` が無い古いスナップショット (同一
  セッション内の Phase 移行時のみ問題になる) は「レイヤ 1 扱い」に倒す。
  読み込み側で `layer ?? 1` の防御を入れる
- **帯の維持**: オブジェクトを足す口は `object:added` の 1 箇所に
  集約されているが、`moveObjectTo` 自体は `object:added` を出さないことを
  確認済みの前提にしている。fabric 更新時はここを再確認
- **選択道具の当て直し**: `applyEntry` は復元後に道具基準で
  selectable を立て直している。§3-2 の `layerFlags` に置き換えるとき、
  選択枠スタイル (`applySelectionStyle`) の呼び順を崩さない
- **性能**: レイヤ導入で描画経路は変わらない (合成は fabric が従来どおり
  z 順で描くだけ)。消しゴム・マーカーの全再描画の重さ (docs/34 §3-2,
  docs/36 §2) にも影響しない

# メモ記法

memo 欄で使える Markdown 記法のメモ(2026-07-15 時点)。
実装は `src/components/MarkdownView.tsx` と
`MermaidDiagram.tsx`。

## 基本(Markdown)

- GFM 対応: 表・タスクリスト・打消し線・裸 URL の自動リンク
  (remark-gfm)
- 単一改行はそのまま改行として表示される(remark-breaks)
- 生 HTML は無効(react-markdown のデフォルト + rehype-sanitize)。
  `<img width="...">` のような HTML タグは書けない
- 本文は最大 10,000 文字

## 画像

- エディタの添付ボタン・貼り付け・ドロップでアップロードすると
  `![](/api/images/<uuid>.png)` が本文に挿入される
- 対応形式: png / jpeg / gif / webp、1 ファイル 10MB まで
- 保存先: DB の `images` テーブル(bytea)。ファイルシステムには置かないため、
  `pg_dump` だけでメモと画像が一緒にバックアップできる
- 画像はクリックで拡大表示される(もう一度クリックか Esc で閉じる)

### 幅指定(独自記法)

alt の末尾に `|数字` を付けると表示幅(px)を指定できる(Obsidian 風)。

```text
![|200](/api/images/xxxx.png)          ← 幅 200px、alt なし
![スクショ|200](/api/images/xxxx.png)  ← 幅 200px、alt「スクショ」
```

- 高さは縦横比を保って自動で決まる
- 末尾が数字でない `|` は通常の alt として扱う(例: `![a|b](...)`)

## タグ(#tag)

本文に `#` から始まる語を書くと**タグ**になる。タグは検索で使え
(`#タグ名` でそのタグの付いたノートだけを絞り込める)、アイテム画面には
タグの一覧が表示される。実装は `src/lib/tags.ts`。

```text
#トランジスタ #2sc1815 #part-a
```

- `#`(半角)または `＃`(全角)の直後に 1 文字以上続けるとタグになる
- タグに使える文字: 文字・数字・`_`・`-`(日本語も可)。数字だけのタグ
  (`#1608` など)も付けられる
- 大小文字・全半角は同一視される(`#NPN` と `#npn` は同じタグ)
- `# 見出し` のように `#` の直後に空白があるものは Markdown 見出しで、
  タグにならない。URL 中の `#fragment` やコード内の `#` も対象外
- タグは本文から自動抽出される(保存時に再計算)。専用の入力欄はない
- 本文中のタグはクリックするとそのタグ検索になる

## プロパティ(hFE=208)

**行全体が `キー=値` だけでできている行**を書くと、その行は部品の**プロパティ**に
なる。タグで検索すると、ヒットしたノートのプロパティが検索結果の上に
**特性表**として並ぶ(同じ種類の部品を並べて比べられる)。
実装は `src/lib/props.ts` と `src/components/PropsTable.tsx`。

```text
#BJT #2sc2712-y
hFE=208 Vf=700mV
```

上のノートは `#BJT` で検索すると、他の `#BJT` のノートと一緒にこう並ぶ。

```text
部品                    device    hFE   Vf
#4951 2SC2712-Y        —         208   700mV
#4908 2SC2166 NPN-BJT  2SC2166   170   625mV
#4339 BJT NPN SS8050   SS8050    300   630mV
```

- **書く位置は自由**。行全体が `キー=値` なら何行目でもよく、複数行に分けても
  まとめて 1 つの部品のプロパティになる(同じキーを 2 回書いたら先に書いたほうが勝ち)
- **散文に混ざった `キー=値` は拾わない**。`実測では hFE=195 だった` のような行は
  行全体が `キー=値` ではないので、ただの本文として扱われる。
  だから本文中で気軽に `=` を使ってよい
- `=` の前後に空白を入れない(`hFE = 208` はプロパティにならない)
- キーは英字で始める(`hFE` `Vf` `Vceo` `device`)。大小・全半角は同一視される
  (`hFE` と `HFE` は同じ列)。表のヘッダには最初に書いた綴りが出る
- 値は空白を含まなければ何でもよい(`208` `700mV` `0.7V` `-40` `120～200`
  `TO-92` `2SC1815`)。表には書いたまま出る
- 箇条書き(`- hFE=208`)や表の中はプロパティにならない。行頭から `キー=値` だけを書く
- コード・数式の中も対象外

### 部品名を列に出す

`Item` に部品名の欄は無いため、表の「部品」列には番号とメモの要約が出る。
部品名を確実に列として出したいときは `device=` に書く。

```text
device=2SC2712-Y hFE=208 Vf=700mV
```

### 単位は値に付ける

`Vf=700mV` のように単位は値へ付ける。半年後の自分が「Vf はミリボルトで書く」規約を
覚えている保証はないので、値に単位があるほうがノート単体で読める。

ヘッダをクリックするとその列で並べ替わる。**単位の換算はしない**ので、
`Vf=700mV` と `Vf=0.7V` が混ざると数値の大小(700 > 0.7)で並ぶ。
同じ列は単位を揃えて書くと期待どおりに並ぶ。

## 数式(KaTeX)

`$...$` で囲むとインライン数式、`$$` だけの行で囲むとブロック数式として
KaTeX で描画される(remark-math + rehype-katex、サーバー側で HTML に変換)。

```text
インライン: 質量エネルギーは $E = mc^2$ で表せる

ブロック:
$$
\int_0^1 x^2 \, dx = \frac{1}{3}
$$
```

- 使える記法は KaTeX がサポートする LaTeX コマンドの範囲
  ([https://katex.org/docs/supported](https://katex.org/docs/supported))
- 通貨表記など数式にしたくない `$` は `\$` とエスケープする。
  同じ段落に `$` が 2 つあると、挟まれた文字列が数式として
  解釈されることがある
- 構文エラーの数式は、書いた内容がそのまま赤字で表示される

## 回路図(CircuiTikZ)

` ```circuitikz ` フェンスを書くと電子回路図として描画される
(TeX を WebAssembly に移植した node-tikzjax をサーバー側で実行)。

````markdown
```circuitikz
\begin{circuitikz}
\draw (0,0) to[R=$R_1$] (2,0) to[C=$C_1$] (2,-2) node[ground]{};
\end{circuitikz}
```
````

- `\begin{circuitikz}` 〜 `\end{circuitikz}` まで自分で書く。
  `\usepackage{circuitikz}` と `\begin{document}` は自動で付く
  (web で見つけた CircuiTikZ の例をほぼそのまま貼れる)
- 部品やオプションは
  [CircuiTikZ のマニュアル](https://ctan.org/pkg/circuitikz)を参照
- ラベルの `$R_1$` は TeX の数式記法がそのまま使える
- 描画は初回だけ 1 秒ほどかかり、以降はキャッシュから即座に表示される
  (本文を書き換えると描き直す)
- 描画が 10 秒を超えると中断される。TeX は無限ループを書けてしまうため
- 1 つのメモに描ける回路図は 8 個まで
- 構文エラーのときは TeX のエラーログが赤枠で表示される

### 描けないもの

`\special{dvisvgm:raw ...}` のように SVG を直接書き込む記法は、
図として想定していない要素 (リンクや外部画像など) を持ち込めるため、
検査に引っかかって描画エラーになる。回路の記法として使うぶんには問題ない。

### 使えるパッケージ

`circuitikz` のほか、`tikz-cd` / `chemfig` / `pgfplots` / `amsmath` /
`amssymb` / `tikz-3dplot` などが読み込まれている。

## mermaid

` ```mermaid ` フェンスを書くと図として描画される
(mermaid v11.16 を動的 import)。

````markdown
```mermaid
flowchart TD
    A --> B
```
````

### サポートする図種

mermaid v11.16 本体に含まれる図はすべて描画できる
(外部プラグインが必要な zenuml を除く)。主なもの:

| 記法(先頭行) | 図 |
| --- | --- |
| `flowchart` / `graph` | フローチャート |
| `sequenceDiagram` | シーケンス図 |
| `classDiagram` | クラス図 |
| `stateDiagram-v2` | 状態遷移図 |
| `erDiagram` | ER 図 |
| `gantt` | ガントチャート |
| `pie` | 円グラフ |
| `gitGraph` | git ブランチ図 |
| `mindmap` | マインドマップ |
| `timeline` | 年表 |
| `quadrantChart` | 4 象限チャート |
| `xychart-beta` | 折れ線・棒グラフ |
| `sankey-beta` | サンキー図 |
| `block-beta` | ブロック図 |
| `packet-beta` | パケット図 |
| `kanban` | かんばんボード |
| `architecture-beta` | アーキテクチャ図 |
| `journey` | ユーザージャーニー |
| `requirementDiagram` | 要求図 |
| `C4Context` など | C4 モデル図 |

### サイズ調整

図の先頭に init ディレクティブを書くとフォントサイズを変えられる
(フォントに合わせて図全体も縮む)。

```text
%%{init: {"themeVariables": {"fontSize": "12px"}}}%%
```

図はコンテナ幅が上限(useMaxWidth: true)なので、
自然サイズより大きく引き伸ばされることはない。

### 制限(securityLevel: "strict")

- `click` によるリンク・コールバックは無効
- ラベル内の HTML はエスケープされる
- `securityLevel` や `maxTextSize` などの secure 設定は
  init ディレクティブから変更できない

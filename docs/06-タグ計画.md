# タグ機能の設計 (Ver2.x)

ノートに複数のタグを付け、タグで検索・絞り込みできるようにする。
利用者向けの使い方は [メモ記法](./メモ記法.md) と
[全文検索の使い方](./05-全文検索の使い方.md) を参照。ここでは設計判断を残す。

## 1. タグ記法 (本文インライン)

タグの正本は**メモ本文**に置く (Obsidian 流のインラインタグ)。専用の
入力欄は設けない。

- `#`(半角)/`＃`(全角)の直後に 1 文字以上のタグ文字が続くものをタグとする
- タグ文字 = Unicode の文字・数字・結合文字と `_` `-`
- 正規化は NFKC + 小文字化 (全文検索の `NormalizerAuto` と挙動を揃える)
- 除外: `# 見出し`(# の直後が空白)、URL 中の `#fragment`(直前が非空白)、
  コードフェンス/インラインコード内
- 数字だけのタグ (`#1608` 等) も許可する。部品のパッケージサイズを
  タグにできると実用的なため。誤爆したら `"#1"` と引用すれば全文検索になる

実装: `src/lib/tags.ts`(`extractTags` / `findTags` / `normalizeTag` /
`parseTagToken`)。DB 非依存の純関数として `search.ts` と同じ流儀でテストする。

## 2. データ設計 — 派生キャッシュ列

`items.tags text[]` に、メモから抽出したタグ(正規化済み)をキャッシュする。
検索は配列包含 `tags @> ARRAY['xxx']` で、GIN インデックス
(`items_tags_idx`)が効く。

正規化テーブル (`Tag` / `ItemTag`) は**採用しない**。タグの正本はメモ本文で、
列は検索用の派生値にすぎない。保存のたびに `extractTags(memo)` で再計算する
(`upsertMemo` / `upsertItem`)。タグ一覧・件数は `unnest(tags)` の集計
(`listTags`)で足りる。将来タグ自体にメタデータ(色など)を持たせたく
なったら、そのとき初めてテーブル化する(YAGNI)。

### PGroonga インデックスとの共存 (注意)

全文検索の PGroonga インデックスは Prisma スキーマで表現できないため、
`prisma migrate dev` は毎回この索引を drift とみなして DROP を生成する。
これをそのまま適用すると `&@` 全文検索が壊れる。マイグレーションは
生成後に手で編集し、DROP を除いて `CREATE INDEX IF NOT EXISTS ... pgroonga`
で張り直す(冪等)。`20260714212059_add_item_tags` がその例。

## 3. 検索文法

既存の DNF パーサ(空白=AND、`OR`/`|`=OR、引用=リテラル)にタグ項を足す。

- 引用されていない `#○○` → タグ項(`items.tags` の完全一致のみ。
  memo/url/itemNo は見ない。「タグとして付けたもの」と「本文に偶然含む
  文字列」を区別できるのがタグ検索の価値なのでフォールバックしない)
- 引用された `"#tag"` → 従来どおりの全文検索リテラル
- 引用しない `#` 単独(タグ名なし)→ 無視

実装: `src/lib/search.ts`(`SearchTerm = text | tag`)、WHERE 組み立ては
`src/lib/items.ts` の `termCondition`。

## 4. タグ名の補完 (検索窓)

`#` を打つと既存タグの候補を出す。事前ロード方式:サーバ(`page.tsx`)で
`listTags()` を 1 回取得してフォームに渡す。キーストロークごとの API 呼び出しは
しない(個人利用でタグ総数は小さい)。将来タグが数千件規模になったら debounce
付き API に切替。

- ロジックは純関数 `src/lib/tagComplete.ts`(文脈判定・前方一致・最長共通
  プレフィックス・置換)に隔離してテストする
- UI は薄いクライアント成分 `src/components/SearchForm.tsx`。素の GET フォームの
  ままなので JS 無効でも検索は動く(プログレッシブエンハンスメント)
- キー操作: `↓`/`↑`+`Enter` で確定、未選択の `Enter` は送信、`Tab` は bash 風
  補完、`Esc` で閉じる。IME 変換中(`isComposing`)はキーを横取りしない

## 5. UI

- アイテム画面: `item.tags` をチップ表示。クリックで `/?q=%23タグ名` へ
- Markdown 表示: 本文中の `#タグ` を検索リンク化する remark プラグイン
  (`src/components/remarkTagLinks.ts`)。mdast の text ノードだけを対象にし、
  コード・既存リンク内は変換しない。表示は元の綴り、リンク先は正規化名

## 6. バックフィル

既存ノートは `scripts/backfillTags.ts` で `tags` を埋める。冪等
(`extractTags` は純関数)。`@updatedAt` を発火させない生 SQL で更新し、
一覧の並び順(更新日順)を保つ。いつでも再実行して整合を回復できる。

```bash
npx tsx scripts/backfillTags.ts
```

## 7. 今回見送り

- ネストタグ (`#部品/抵抗`): 後から `/` を許可文字に足してバックフィル
  再実行するだけで導入できる。最初から入れると検索・UI が複雑になる
- タグ一覧/件数ページ、CodeMirror でのタグ補完・ハイライト:
  `listTags()` を土台に別タスクで

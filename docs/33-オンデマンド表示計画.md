# 検索一覧のオンデマンド表示 (Ver2.x)

README (親 repo) の TODO「ノート：一覧をオンデマンド表示？」の設計と実装。
「前へ / 次へ」のページ送りを廃止し、一覧の末尾が見えたら続きが自動で
伸びる方式に置き換えた。実装済み (2026-07-20)。

## 1. 方式の選定

- **「次へ」との併設 (選択式) はしない**。一人用アプリに表示方式の設定は
  YAGNI。代わりに「さらに表示 (残り n 件)」を**見えるリンク**として置き、
  それが画面に入ったら自動発火する形にした。observer が効かない環境では
  普通のリンクとして押せるので、フォールバックが「次へ」の役割を吸収する
- 無限スクロールの一般的な難点はこのアプリでは薄い:
  総件数は一覧の先頭に常に出ている・フッターは固定の下部バー
  (docs/31) なので「末尾に届けない」問題が無い

## 2. `?page=N` の意味論変更 (サーバ側)

`searchItems(query, page)` の page N を「N ページ目の 20 件」から
**「1〜N ページ目の累積」** (`LIMIT N*PAGE_SIZE`、OFFSET なし) に変えた。

- **クライアントに蓄積 state を持たない**のが狙い。読み込み済みリストを
  クライアントで積む方式は、打鍵ごとの URL 書き換え (docs/11 §3) と
  「クエリが変わったら蓄積をリセット」の同期が必要になり、state 同期
  バグの温床になる。累積読み込みならサーバが毎回全体を返すだけ
- 「URL が正」を維持: 共有・再読込・戻るで同じ表示が再現する。
  旧 URL の `?page=3` は「3 ページ目だけ」→「1〜3 ページ目」に変わるが
  壊れはしない
- OFFSET 型の古典的バグ (読み込みの合間にデータが変わると行が重複/欠落)
  が原理的に起きない。毎回先頭からのスナップショットなので、カーソル
  (keyset) 方式を導入する必要が無い
- 転送量・DB 負荷は N ページ目で 1〜N 全件ぶんになるが、個人規模
  (数百〜数千件、PAGE_SIZE=20) では誤差 (docs/15 §2-2 と同じ判断)
- **前提のバグ修正**: 更新順が `ORDER BY updated_at DESC` のみで同時刻の
  行の並びが不定だった (docs/15 §2-2 で指摘済み)。累積の引き直しでは
  読み込みのたびに行が入れ替わって見えるため、`, item_no ASC` の
  タイブレークを先に入れた

## 3. AutoLoadMore (クライアント側)

`src/components/AutoLoadMore.tsx`。唯一の新規クライアントコード。

- 一覧末尾に「さらに表示 (残り n 件)」の `PendingLink` を置き、
  IntersectionObserver (rootMargin 200px で少し先読み) が見えたら
  `router.replace(次ページURL, { scroll: false })` する
- **replace であって push ではない**: ページ送りは「同じ検索を続きまで
  見る」操作で、戻るボタンで 1 ページずつ縮んでほしくない
  (SearchNav の打鍵と同じ扱い)
- 全件表示し切ったら何も出さない。「n / m ページ」の表示も廃止
  (件数は先頭にある)
- **センチネルに `overflow-anchor: none` が必須** (E2E 検証で発覚)。
  末尾までスクロールした状態で行がセンチネルの上に挿入されると、
  ブラウザのスクロールアンカリングがセンチネルを基準に viewport を
  追従させ、観測が切れずに全ページを連鎖読み込みしてしまう。
  アンカー候補から外せば viewport は既存行に留まり、1 スクロール =
  1 ページで止まる (Playwright で 20→40→60 行の停止を確認済み)
- 画像ビュー (docs/32) は画像持ちノートだけがタイルになるため、画面が
  埋まらない間は自動で読み進む。これは「画面が埋まるまで読む」動作
  として意図どおり (個人規模なら全件でも軽い)

## 4. 既存機能への影響 (確認済み)

- **一括操作 (タグ・ゴミ箱)**: `ItemList` の hidden `page` は「表示中の
  最大ページ」になり、`parseBackUrl` の戻り先 `/?page=N` が「展開して
  いた分まで戻る」に自然に変わる。選択モードはページ内 20 件ではなく
  **読み込んだ全件から選べる**ようになった (改善)
- **/item 前後ナビ計画 (docs/15 §2)**: 「一覧へ戻る =
  `buildSearchUrl(query, floor(index/PAGE_SIZE)+1)`」は、新しい意味論では
  「その項目を含む位置まで展開した一覧」になる。アンカーへのスクロールを
  足せば計画のままより自然に戻れる。計画の修正は不要
- **表示モード (docs/23 / docs/32)**: compact / card / masonry いずれも
  リストが伸びるだけ。React の差分描画で既存行の DOM は動かない

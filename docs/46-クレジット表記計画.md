# 46 クレジット表記計画 (ハンバーガーメニュー → クレジットページ)

外部 API の利用に伴うクレジット/帰属表示を、ハンバーガーメニューから開く
専用ページに集約する計画。Yahoo!デベロッパーネットワークはクレジット表示が
**義務**で、規定 HTML の改変・極端な縮小が禁止されている。

## 1. 義務の中身

### 1-1. Yahoo (必須・規定 HTML)

- 対象: Yahoo!デベロッパーネットワーク提供 API を使う**すべて**のサイト・アプリ
  (本アプリは Yahoo!ショッピングで JAN 商品情報を引く)。
- 位置: 「アプリケーションの下部に配置」。
- 規定: 下記 HTML の**そのまま**。改変・CSS での色変更・極端な縮小は禁止。
  規定 HTML は折り返せないため、この行だけ行長 lint を無効化している。

  <!-- markdownlint-disable MD013 -->

  ```html
  <!-- Begin Yahoo! JAPAN Web Services Attribution Snippet -->
  <span style="margin:15px 15px 15px 15px"><a href="https://developer.yahoo.co.jp/sitemap/">Webサービス by Yahoo! JAPAN</a></span>
  <!-- End Yahoo! JAPAN Web Services Attribution Snippet -->
  ```

  <!-- markdownlint-enable MD013 -->

  (英語版は「Web Services by Yahoo! JAPAN」。実装前に
  <https://developer.yahoo.co.jp/attribution/> で最新の規定 HTML を再確認する。)

### 1-2. 楽天 (要求があれば)

楽天ウェブサービス規約 第 13 条: 会社が要求する場合、Product Information が
Group によって供給されている旨を表示。Rakuten Developers の帰属文言/バナーを
併記しておく。

### 1-3. openBD / NDL サーチ (任意・出典として)

義務は緩いが、出典として併記すると親切。書誌が openBD / NDL 由来であることを一言。

## 2. 方針: 専用ページ + ハンバーガーから導線

**「クレジット / About」ページを 1 枚作り、`HeaderMenu` から辿らせる。**
既存の IA (ログ・パスキー・インポートはハンバーガーから開くページ) に合わせる。

### 2-1. なぜフッター常時表示にしないか

本アプリはスマホ主戦場で下部操作バー (`BottomActionBar`) が画面下にいる。
Yahoo の「下部に配置」をグローバルフッターで満たすと操作バーと競合する。
専用ページで規定 HTML を**読める大きさ**で出せば「極端に小さくしない」の
趣旨を満たせる。

### 2-2. 「下部に配置」の解釈リスク (要検証)

厳格に読むと、API 結果が出るエディタ画面に常時クレジットを置く解釈もありうる。
ハンバーガー内ページは常時表示ではない。心配なら Phase 2 で結果表示画面
(エディタ) のフッターにも規定 HTML を出す二段構えにする (§5)。まず About
ページで開始する。

## 3. 変更点 (実装時の当たり所)

### 3-1. 新規ページ: `src/app/about/page.tsx` (パス名は要検討)

- `settings/passkeys/page.tsx` を雛形にする (Server Component、`metadata.title`
  はサイト名を付けない — root layout の template が付ける)。
- **デモでも表示する**。設定系 (パスキー等) と違い、クレジットは
  `notFound()` に倒さない — デモでも Yahoo API 利用の可能性があり、帰属表示は
  むしろ常に出してよい。
- 内容: 見出し + 各社クレジット。Yahoo は規定 HTML を**改変せず**そのまま置く
  (`dangerouslySetInnerHTML` か、規定通りの JSX で。span の inline style も
  規定のまま保持する)。楽天・openBD・NDL の帰属を続ける。
- 外部リンクは `target="_blank" rel="noopener noreferrer"` (既存の GitHub リンク流儀)。

### 3-2. メニュー項目: `layout.tsx`

- `HeaderMenu` の children に「クレジット」項目を追加。`Link href="/about"` +
  `HEADER_MENU_ITEM_CLASS`。
- **出す位置**: ログイン/未ログインの両方で見せてよい (帰属表示は誰にでも
  出してよい情報)。少なくともログイン中は他項目と並べる。未ログインでも
  出すか、ログイン中だけかは、メニューの混み具合を見て決める。
- **デモでも隠さない** (`!isDemo` の内側に入れない)。

### 3-3. アイコン: `MenuIcons.tsx`

- 行頭アイコンを 1 つ足す (情報の "i" 丸、またはハートなし。既存の
  `StrokeIcon` + `TINT` の作法に合わせ、`size-5`・currentColor・専用色クラス)。
- ライブラリは足さない (既存方針)。inline SVG で "i" 丸を 1 個描く。

## 4. 文言・レイアウト案

- 見出し: 「クレジット」または「このアプリについて」。
- リード文 1 行: 「本アプリは以下の Web サービスを利用しています。」
- 各社ブロック: サービス名 + 規定クレジット/リンク。Yahoo は規定 HTML を
  そのまま。楽天は「Supported by Rakuten Developers」等の帰属 + リンク。
- ページ全体は `settings/passkeys` と同じ `space-y-6` の縦積みで十分。

## 5. Phase 2 (実施済み → docs/47): エディタ画面のフッターにも Yahoo クレジット

「下部に配置」を厳格に満たしたい場合のみ。JAN 取得を実際に使う編集画面
(`MemoEditor` 近辺) の下部に規定 HTML を常設する。About ページと**同じ
規定 HTML** を共有コンポーネント化して二重管理を避ける。

→ [47-エディタクレジット常設計画.md](47-エディタクレジット常設計画.md) で実施。
規定 HTML は `YahooAttribution.tsx` に集約し、/about と `/edit/[itemNo]` の
下部の両方から使う。

## 6. テスト観点

- `/about` が 200 で開き、Yahoo 規定 HTML (href・span・文言) が**そのまま**
  含まれること (改変検知のためスナップショットか文字列一致で固定)。
- デモでも `/about` が開けること (`settings/*` と違い 404 にしない)。
- ハンバーガーに「クレジット」項目が出て `/about` へ遷移すること。

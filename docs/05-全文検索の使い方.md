# 全文検索の使い方

このアプリ (`qr.tommie.jp`) の検索窓で使える検索方法をまとめる。
実装は PGroonga による全文検索 (`src/lib/items.ts` / `src/lib/search.ts`)。
実装の背景・設計判断は [04-全文検索計画.md](./04-全文検索計画.md)、
論理演算 (AND / OR / NOT / 括弧) の細かい規則は
[16-検索の論理演算.md](./16-検索の論理演算.md) を参照。

## 1. 何が検索対象か

トップ画面の検索窓に入力した語は、次の 3 つを横断して探す。

| 対象 | 一致方式 | 補足 |
| --- | --- | --- |
| メモ本文 (`memo`) | 全文検索 (`&@`) | 日本語バイグラム。文中のどこにあっても一致 |
| URL (`url`) | 全文検索 (`&@`) | 同上 |
| 部品番号 (`itemNo`) | 前方一致 (`ILIKE 'xxx%'`) | 先頭からの一致。大小文字は無視 |

1 つの語につき「メモ **または** URL **または** 部品番号のいずれかに一致すれば
ヒット」という扱い。これが後述の「暗黙の OR」にあたる。

### 表記ゆれの吸収 (正規化)

PGroonga の `NormalizerAuto` により、次は同一視される。

- 全角/半角: `２ＳＣ１８１５` ↔ `2SC1815`
- 大文字/小文字: `NPN` ↔ `npn`

そのため `2sc1815` と打っても `２ＳＣ１８１５` を含むメモがヒットする。

## 2. AND 検索 (すべての語を含む)

**語をスペースで区切る**と AND 検索になる。半角スペースでも全角スペースでもよい。

```text
抵抗 1608
```

→「抵抗」**かつ**「1608」の両方を含むものだけがヒットする。

- 区切りは半角スペース・全角スペース・タブ・改行のいずれでも可
- 連続スペースや前後の余分なスペースは無視される
- 同じ語を重複して入れても 1 回として扱う
- AND できる語数は最大 10 語 (`MAX_SEARCH_TERMS`)。超過分は切り捨て

各語ごとに「メモ or URL or 部品番号」を評価し、**語と語の間が AND** になる。
たとえば `抵抗 1608` は次の意味になる。

```text
(memo に「抵抗」 or url に「抵抗」 or itemNo が「抵抗」で始まる)
  AND
(memo に「1608」 or url に「1608」 or itemNo が「1608」で始まる)
```

## 3. OR 検索 (どれかを含む)

### 3-1. 暗黙の OR — 1 語は 3 フィールドを OR で探す

前述のとおり、1 語は「メモ or URL or 部品番号」を横断する。
つまり `1815` と 1 語だけ打てば、**メモに 1815 を含むもの / URL に 1815 を含むもの /
部品番号が 1815 で始まるもの**が OR でまとめてヒットする。
フィールドを跨いだ OR は意識しなくても効いている。

### 3-2. 語と語の OR — `OR` または `|` で区切る

「抵抗 OR コンデンサ」のように**別々のキーワードを OR したい**場合、
`OR` (大文字小文字は区別しない) か `|` (パイプ) で区切る。

```text
抵抗 OR コンデンサ
抵抗|コンデンサ
```

→「抵抗」**または**「コンデンサ」のどちらかを含むものがヒットする。

- `OR` は前後を空白で区切った**単独の語**のときだけ演算子になる
  (`or` / `Or` でも同じ。`DOOR` などの中の文字列は演算子にならない)
- `|` は前後の空白の有無を問わず区切る (`A|B` でも `A | B` でもよい)
- 3 語以上も可: `A OR B OR C` / `A|B|C`
- **全角のまま打ってもよい** (`ＯＲ` / `｜`)。日本語入力を切り替える必要はない

#### 演算子の優先順位

強く結合する順に **`!` (NOT) > 空白 (AND) > `OR`**。SQL や Google 検索と同じ
感覚でよい。そのため次は同じ意味になる。

```text
抵抗 1608 OR コンデンサ
```

→ `(「抵抗」かつ「1608」) または 「コンデンサ」` と解釈される。
この順番を変えたいときは括弧で囲む (§3.7)。

内部では式を組み立て直してから SQL の `AND` / `OR` / `NOT` に写し、各語は
これまでどおり素のキーワードとして `&@` に渡す。演算子構文を PGroonga に
直接渡さないため、構文エラーやエスケープ漏れは起きない
([04-全文検索計画.md](./04-全文検索計画.md) §3-3)。

#### `or` / `|` を語として検索したいとき

`OR` や `|` は演算子として予約されている。これらを**文字どおり検索したい**
ときはダブルクォートで囲む。

```text
"or"       ← 語「or」を検索 (OR 演算子にならない)
"A|B"      ← 語「A|B」を検索 (パイプで区切られない)
A "or" B   ← 「A」かつ「or」かつ「B」の AND
```

## 3.5. タグ検索 (#tag)

メモ本文に書いた**タグ** (`#トランジスタ` など) で絞り込める。タグ記法は
[メモ記法](/docs/memo) を参照。検索窓では、引用されていない `#○○` が
タグ検索になる。

```text
#トランジスタ
```

→ タグ `トランジスタ` の付いたノートだけがヒットする (本文に文字列として
「トランジスタ」を含むだけのノートは**ヒットしない**。タグとして付けたものだけ)。

- 大小文字・全半角は同一視される (`#NPN` = `#npn`)
- タグも論理演算できる (空白 = AND、`OR` / `|` = OR、`!` = NOT、`()` でグループ化)

```text
#tag1 #tag2      ← tag1 かつ tag2
#tag1 | #tag2    ← tag1 または tag2
#tag1 !#tag2     ← tag1 かつ tag2 でない
#トランジスタ 2SC1815   ← タグと全文検索の混在も可
```

### タグ名の補完

検索窓で `#` を打つと、既存タグの候補が出る。`#t` のように途中まで打つと
前方一致で絞り込まれる。

- `↓` / `↑` で候補を選び `Enter` で確定 (候補を選んでいないときの `Enter` は
  通常の検索送信)
- `Tab` は bash 風の補完 (候補が 1 つなら確定、複数なら共通部分まで補完)
- `Esc` で候補を閉じる。日本語入力の変換中はキー操作を邪魔しない

### `#` を文字として検索したいとき

`#` はタグの記号として予約されている。`#` やタグ名を**文字どおり検索**
したいときはダブルクォートで囲む。

```text
"#"        ← 記号「#」を全文検索
"#tag1"    ← 文字列「#tag1」を全文検索 (タグ検索にならない)
```

引用しない `#` 単独 (タグ名なし) は無視される。

## 3.6. NOT 検索 (`!` で除外)

**語の前に `!` を付けると、その語を含むものを除外**する。全角の `！` も同じ。

```text
#bjt !#npn
```

→ タグ `bjt` が付いていて、**かつ** タグ `npn` が付いて**いない**ものだけ。

- `!` と語の間はスペースがあってもなくてもよい (`!#npn` も `! #npn` も同じ)
- `!` は**直後の 1 つ**だけに掛かる。`!#npn #bjt` は
  「npn でない **かつ** bjt」であって、bjt までは否定されない
- テキスト語の否定は「メモにも URL にも部品番号にも含まない」の意味になる
- 否定だけのクエリも書ける。`!#覚えた` → タグ `覚えた` が付いていない全ノート
- `!` を文字として検索したいときはダブルクォートで囲む (`"!"` / `"!foo"`)

## 3.7. 括弧でグループ化

`()` で囲むと、その中身がひとかたまりの条件になる。全角の `（）` も同じ。

```text
#bjt !(#npn OR #pnp)
(抵抗 OR コンデンサ) 1608
```

→ 上は「bjt で、npn でも pnp でもないもの」、
下は「(抵抗 または コンデンサ) かつ 1608」。

括弧がないと「空白 (AND) は OR より強い」ため、`抵抗 OR コンデンサ 1608` は
`抵抗 または (コンデンサ かつ 1608)` になってしまう。**OR を先に評価させたい
ときに括弧を使う。**

- 括弧の前後にスペースは要らない (`#bjt(!#npn)` でもよい)
- 何重にでも入れ子にできる
- 括弧を文字として検索したいときはダブルクォートで囲む (`"(株)"`)

### 打ち間違えても止まらない

エラーにはせず、それらしく解釈する。

| 打ち間違い | 扱い |
| --- | --- |
| `(抵抗 OR コンデンサ` (閉じ忘れ) | 末尾で閉じたものとみなす |
| `抵抗) 1608` (余分な `)`) | その `)` を無視 |
| `抵抗 !` (相手のいない `!`) | その `!` を無視 |
| `()` (空の括弧) | 無視 |

## 3.8. URL でクエリを直接組み立てる

検索窓は使わず URL パラメータ `q` に語を渡してもよい (検索窓と同じ文法。
スペースは AND、`OR` / `|` は OR、`!` は NOT)。

```text
https://qr.tommie.jp/?q=抵抗%201608
https://qr.tommie.jp/?q=抵抗%201608&sort=itemNo&page=2
https://qr.tommie.jp/?q=抵抗%20OR%20コンデンサ
https://qr.tommie.jp/?q=%23bjt%20!%23npn
```

- `q` — 検索語 (スペース AND・`OR`/`|` で OR・`!` で NOT・`#○○` でタグ)
- `sort` — `updated` (更新順・既定) / `itemNo` (番号順)
- `page` — ページ番号 (1 ページ 20 件)

ブックマークしておけば、よく使う検索をワンクリックで再実行できる。

## 4. 並び順とページング

- 既定は **更新順** (`sort=updated`, 新しい順)。番号順 (`sort=itemNo`) に切替可
- 関連度 (スコア) 順は**採用していない**。小規模データでは PGroonga の
  インデックスが使われず score が 0 になるため
  ([04-全文検索計画.md](./04-全文検索計画.md) §3-4)
- 1 ページ 20 件 (`PAGE_SIZE`)
- 検索窓を空のまま検索すると、全件を並び順どおりにブラウズできる

## 5. 早見表

| やりたいこと | 入力例 | 結果 |
| --- | --- | --- |
| 1 語で横断検索 | `1815` | メモ/URL/番号のどれかに一致 (暗黙 OR) |
| すべての語を含む (AND) | `抵抗 1608` | 両方を含むものだけ |
| どれかを含む (OR) | `抵抗 OR コンデンサ` | どちらかを含むもの |
| どれかを含む (OR・記号) | `抵抗\|コンデンサ` | 同上 (`\|` でも OR) |
| AND と OR の混在 | `抵抗 1608 OR コンデンサ` | `(抵抗 かつ 1608)` または `コンデンサ` |
| 除外する (NOT) | `#bjt !#npn` | bjt が付き、npn が付かないもの |
| まとめて除外 | `#bjt !(#npn OR #pnp)` | bjt で、npn でも pnp でもないもの |
| OR を先に評価 | `(抵抗 OR コンデンサ) 1608` | `(抵抗 または コンデンサ)` かつ `1608` |
| タグで絞る | `#トランジスタ` | そのタグの付いたノートだけ |
| タグの AND / OR | `#tag1 #tag2` / `#tag1\|#tag2` | 両方 / どちらか |
| `#` を文字として検索 | `"#tag1"` | ダブルクォートでタグ検索を無効化 |
| `or` / `\|` / `!` / `()` を語として検索 | `"or"` / `"!"` | ダブルクォートで演算子を無効化 |
| 全角で打っても半角一致 | `２ＳＣ１８１５` | `2sc1815` を含むメモに一致 |
| 演算子を全角で打つ | `#a｜#b` / `#a ＯＲ #b` / `！#b` / `（#a #b）` | 半角と同じに扱う |
| 大小文字を無視 | `npn` | `NPN` を含むメモに一致 |
| 部品番号で絞る | `R100` | `R100…` で始まる番号に前方一致 |
| 全件ブラウズ | (空欄で検索) | 並び順どおり全件 |

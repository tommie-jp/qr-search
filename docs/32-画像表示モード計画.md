# 画像表示モード計画 (Ver2.x)

表示モードに 3 つ目の「画像 (image)」を足す。ページ内のノート本文に貼られた
自前画像だけをピンタレスト風の masonry で敷き詰め、写真からノートを探す
入口にする。1 ノートに複数枚あれば全部並べ、画像の無いノートは出さない。

前提はいつもどおり: 素のフォーム / リンク / Server Action で JS 無効でも動く
設計は壊さない。切替は既存の cookie フォーム送信のまま
(docs/23-検索結果表示モード計画.md §4-5)。

## 1. 画像は行優先の CSS Grid で敷き詰める

`grid grid-cols-[repeat(auto-fill,minmax(min(10rem,100%),1fr))] gap-2` +
タイルの `<img>` に `aspect-square object-contain` (ImageMasonry.tsx)。JS の
masonry ライブラリは足さない — 必要なのは「画面幅に応じて隙間なく敷き詰める」
ことだけで、そのために依存とバンドルを増やす釣り合いが取れない (MenuIcons が
アイコンライブラリを足さなかったのと同じ判断)。

- **列数は指定しない**。列幅 10rem を基準に画面が決める — card グリッドの
  auto-fill と同じ思想 (docs/23 §1)。スマホで 2 列、PC の広幅で 5〜6 列。
  `min(10rem,100%)` を下限にするのは card と同じ罠 (器が 10rem より狭い端末で
  列が広がり横スクロールが出る) を避けるため (ItemList.tsx)。
- タイルの枠は正方形 (`aspect-square`) に固定するが、絵は `object-contain` で
  **全体を余白付きで見せる** (切り抜かない)。余白は地色 (`bg-gray-100`) が埋め、
  これがレターボックスと読み込み前の場所の気配を兼ねる。器 672px (max-w-2xl)
  ÷ 3 列 ≒ 208 CSS px のタイルを高 DPR で賄うため、サムネは 384px の箱に縦横比
  維持で収める (thumbnail.ts, fit: 'inside')。
- 経緯: 当初は絵も正方形に切り抜いて覆っていた (`object-cover` + サムネ
  `fit: 'cover'`)。だが「写真を切り取らず全体を見たい」という要望で contain 表示
  に変えた (余白は許容)。cover 時代に横長写真が甘く見える問題を避けるため
  サムネを cover 保存にしていたが、タイルが contain になったことでその理由は
  消えたので、サムネも縦横比維持 (inside) に戻した。生成パラメータを変えたので
  既存サムネは作り直しが要る (scripts/backfillThumbs.ts --force) うえ、配信 URL
  は 1 年 immutable なので thumbUrl に版 (`&v=`) を混ぜてキャッシュを割る (v3)。

### CSS multi-column から Grid に変えた経緯

当初は Pinterest 風の masonry を狙って `columns-[10rem]` +
`break-inside-avoid` で組んでいた。しかし multi-column は**左の列を上から下へ**
埋めるため、1,2,3 が縦に並ぶ。1 ページ 20 件を眺めるだけなら許容と当初は
判断したが、オンデマンド表示 (docs/33) を入れて末尾にページを継ぎ足すように
なると 2 つの不都合が顕在化した:

- 継ぎ足すたびに全タイルが列間で再配置され、既に見ていた並びがずれる。
- 「番号順に横へ (1,2 / 3,4)」という素直な期待と縦詰めが食い違う。

そこで行優先で埋まり、追記しても既存タイルが動かない CSS Grid に変更した。
寸法は DB に無く読み込みで高さが決まるので、枠を `aspect-square` で正方形に
固定して行の高さのガタつきとレイアウトシフトを消す。絵は `object-contain` で
全体を見せ、余白は地色が埋める (タイルごとに絵の比率で高さを変える真の masonry
は DB に寸法列を持つまで見送り。docs/32 §3 の将来課題と同じ)。
コンポーネント名 `ImageMasonry` は経緯を残すため据え置く。

## 2. タイルは「画像 1 枚 + compact 相当の 2 行キャプション」

描画は ItemList から ImageMasonry へ丸ごと委譲する。ItemRow を拡張しない
のは、あちらの仕掛け (選択モードのチェックボックス差し込み、card の本文
プレビュー) までタイルに背負わせると両方が読めなくなるため。ただし
「#番号 タイトル / タグ」の 2 行キャプションは compact 表示 (docs/23) に
揃えたいので、その部分の作りは ItemRow から借りる。

- 画像の列挙は `allImageNames()` (memoImages.ts)。画像検索の索引づくりと
  同じ抽出規則 (出現順・重複除去・コードフェンス除外・自前画像のみ) を
  そのまま使う。URL モードのノートは memo が空なので自然に落ちる。
- タップでノート詳細 `/item/<itemNo>` へ。画像は入口であって、その場で
  拡大するビューアではない (拡大は詳細ページの ZoomableImage が持っている)。
- 画像の下に **compact 相当の 2 行** を添える (要望): 1 行目 `#番号` +
  タイトル (`memoSummary`) を `truncate` で 1 行に、2 行目にタグ。1 ノートに
  複数枚あれば各タイルに同じキャプションが繰り返し付く (タイル = 画像 1 枚の
  原則は維持)。
- **stretched link を ItemRow から借りる。** タグはタグ検索という別の行き先を
  持つのでタイル全体を 1 本の `<a>` で包めない (リンクの入れ子)。画像 +
  1 行目をノート詳細への `<Link>` にし、その `::after` を枠いっぱいに広げて
  当たり判定だけを大きくする。タグは `relative z-10` で膜の上に出す。
- 当初あった**左下の `#番号` 焼き込みバッジは廃止**。番号がキャプション
  1 行目に出るので、二重表示になり役目を終えた。これに伴い `<img>` の `alt`
  は装飾扱いの `""` に戻す (中身はすぐ下のキャプションが説明する)。
- **next/image は使わない** — 画像 API はログイン必須で、optimizer が
  Cookie 無しで取りに行き 401 になる (docs/23 §2.1、ItemRow.tsx)。素の
  `<img loading="lazy">` + `?thumb=1`。原寸 (数 MB) は引かない。
- 寸法は DB に無いので width/height 属性を出せず、読み込みで高さが決まる
  (layout shift)。`bg-gray-100` の地色と lazy で緩和して許容。気になったら
  Image テーブルに寸法列を足して保存時に埋める (将来課題)。

## 3. 切替は 1 スロットの循環トグル (小→大→画像→小)

3 値になってもセグメントには戻さず、下部バーの 1 スロットで循環する
(BottomActionBar.tsx)。5 スロット構成を崩さないため。ラベルとアイコンには
**現在の値**を出す原則 (docs/31 §3-4) はそのまま。行き先は aria-label
(「押すと◯に切替」) に乗せる。

- アイコンは ImageViewIcon (高さ不揃いのタイル 4 枚 = masonry の形) を
  新設。GridViewIcon (均等 2×2) との差は高さの不揃いだけに抑えて「表示」
  スロットの仲間だと判るようにする。ImageSearchIcon の流用は不可 — 隣の
  「画像検索」スロットと同じ絵になり、色と形で狙えなくなる (docs/31 §11-1)。
- `parseViewMode` は 'image' を受けるだけで、知らない値→compact の畳み込み
  (docs/23 §4) は変えない。**setViewModeAction に revalidatePath を
  足さないこと** (docs/23 §5.1 の EACCES の罠) も変わらず。
- ItemRow には 'image' が流れない。`RowViewMode = Exclude<ViewMode, "image">`
  で型ごと締め出してある (ItemRow.tsx)。

## 4. ページ割りはノート単位のまま

1 ページ = ノート 20 件は変えず、サーバのクエリも触らない。画像モードは
「いま見ているページの画像を敷き詰める」だけで、検索状態 (何を見ているか)
には手を入れない — 表示モードが cookie であって URL でない理由 (docs/23 §4)
と同じ線引き。

その代償として、ページ内の 20 件に画像持ちが 1 件も無いページが起こり得る。
そこでは「このページには画像がありません」を出す (ページ送りは残るので
次のページへは進める)。画像持ちノートだけでページを詰め直す案は、クエリの
分岐とページ番号の意味が他モードとずれる (同じ 3 ページ目が別の 20 件を
指す) ので採らなかった。

0 件 (検索ヒットなし) は別物で、該当なしの案内と新規登録/ゴミ箱の導線
(docs/10, docs/12) を画像モードでも失わない。

## 5. 選択モード中は compact の一覧に畳む

画像モードで下部バーの「選択」に入ると、masonry ではなく compact の一覧に
チェックボックスを出す (ItemList.tsx の rowView)。タイルは画像単位で
ノート単位ではなく、画像の無いノートがタイルに出ない — 「このページの
ノートから選ぶ」操作の対象と画面が食い違うため。選択を抜ければ masonry に
戻る。BulkTagToolbar / form の仕掛けもそのまま使い回せる。

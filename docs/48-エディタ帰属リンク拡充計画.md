## 1. 何を変えるか (47 §3-3 の決定を覆す)

docs/47 §3-3 は「エディタ下部は Yahoo の規定クレジットだけ。義務があるのは
Yahoo だけで、全社を並べる理由はない」とした。今回これを覆し、**このアプリが
使っている Web サービスすべてへの導線**をこの位置に置く。

- 目的は義務 (Yahoo) から一歩広げて、**出典の明示**にする。書名・著者・書影・
  商品情報がどこ由来かを、結果が現れるその画面で辿れるようにする。
- Yahoo だけは規約で規定 HTML の改変が禁止なので**そのまま**。他社は義務では
  ないので、体裁を揃えた普通のリンクでよい。

## 2. どのサービスを・どのページへ (適切なリンク先)

実際に使っているのは 4 つ (Yahoo + 3)。各サービスの「適切なページ」は下記。

| サービス | 用途 | リンク先 (適切なページ) |
| -------- | ---- | ---------------------- |
| Yahoo!ショッピング | JAN 商品情報 | 規定どおり <https://developer.yahoo.co.jp/sitemap/> (改変不可) |
| 楽天ブックス書籍検索API | 書影 | <https://webservice.rakuten.co.jp/documentation/books-book-search> |
| openBD | 書誌 | <https://openbd.jp/> |
| 国立国会図書館サーチ | 書誌 | <https://ndlsearch.ndl.go.jp/> |

補足:

- **楽天**は、汎用トップ (webservice.rakuten.co.jp) ではなく、実際に叩いている
  「楽天ブックス書籍検索API (version 2017-04-04)」のドキュメントを指す。
  コードのエンドポイント `BooksBook/Search/20170404`
  ([rakutenBooks.ts](../src/lib/rakutenBooks.ts)) と版まで一致する。
- **国会**は利用者向けにはサーチのトップ
  (<https://ndlsearch.ndl.go.jp/>) が素直。API 由来を強調するなら
  「API 仕様の概要」(<https://ndlsearch.ndl.go.jp/help/api/specifications>)
  も候補。**推奨はトップ**。
- **openBD** はトップにそのまま API 説明があるのでトップでよい。
- **要判断 (openBD の扱い)**: 依頼の明示は「楽天・国会」の 2 つ。ただし
  openBD も現に書誌で使っており、/about でも既に帰属している。**足並みを
  そろえて openBD も入れる**のを推奨 (使っている以上、出典として出すのが自然)。
  外すなら 3 行 → 2 行にするだけ。

## 3. リンク先 URL を 1 か所に集約する (ドリフト防止)

いま /about ([about/page.tsx](../src/app/about/page.tsx)) が各社リンクを
JSX 直書きで持っている。エディタ下部にも同じ URL を並べると、二重管理で
片方だけ古くなる。**URL とラベルの単一の出どころ**を作る。

- 案: `YahooAttribution.tsx` を、Yahoo 規定 HTML **＋** 他社リンク配列を持つ
  帰属フッターに広げる (名前も `AttributionFooter.tsx` へ改める)。
  - Yahoo は従来どおり規定 HTML をそのまま (改変不可の制約は不変)。
  - 他社は `const SERVICES = [{ label, href, note }...]` を map してリンク化。
  - /about もこの `SERVICES` を import して使い、URL の出どころを 1 つにする
    (/about は説明文が厚いので、リンク部分だけ共有すればよい)。
- こうすると「エディタ下部」と「/about」でリンク先が必ず一致する。

## 4. 体裁 (エディタ下部)

- Yahoo の規定スニペットを先頭に (現状のまま)。その下に他社を小さめの
  リンク列で並べる。**極端に縮小しない** (Yahoo の「極端に小さくしない」に
  合わせ、他社も同等の可読性にそろえる)。
- 文言案: 「データ提供:」に続けて
  「楽天ブックス書籍検索API ／ openBD ／ 国立国会図書館サーチ」をリンクで。
- 外部リンクは `target="_blank" rel="noopener noreferrer"` (既存の about・
  GitHub と同じ流儀)。Yahoo 規定 HTML には足さない (規定に無いため)。
- form の外・ページ末尾に置く点は 47 のまま (sticky「更新」と重ならない)。

## 5. /about との役割分担 (整理)

- /about: 全社を説明文つきで挙げる総覧。ここは従来どおり厚い版。
- エディタ下部: 結果が出るその場での出典導線。Yahoo 規定 + 他社リンクの
  コンパクト版。
- 両者はリンク先 URL (`SERVICES`) を共有し、文言・レイアウトだけ各所で持つ。

## 6. テスト観点

- `AttributionFooter` (旧 YahooAttribution): Yahoo 規定 HTML が一字一句
  そのまま出続けること (既存テストを維持)。加えて、楽天 (books-book-search)・
  openBD・国会 の各 href が出ること。
- /about: 同じ URL 集合が出続けること (共有 `SERVICES` の回帰)。
- 楽天のリンク先が API ドキュメント (books-book-search) を指すこと
  (汎用トップに戻っていないことの固定)。
